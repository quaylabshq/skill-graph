# Copy & Writing Standards

## Voice & Tone

- Use active voice — "Save your changes" not "Changes will be saved"
- Use second person — "your", "you"
- Use `&` where space is constrained (tab labels, breadcrumbs)

## Formatting

- Title Case for headings and button labels
- Use numerals for counts — "3 items" not "three items"

## Buttons & Actions

- Use specific button labels — "Save Project" not "Submit"
- Labels should describe what happens, not the UI element

## Error Messages

- Error messages must include the fix — "Enter a valid email address" not just "Invalid input"
- Be specific about what went wrong and how to resolve it
