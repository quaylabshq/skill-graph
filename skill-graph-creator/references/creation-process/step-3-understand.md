# Step 3: Understand the Skill with Concrete Examples

Skip this step only when the skill's usage patterns are already clearly understood. It remains valuable even when working with an existing skill.

## Purpose

Clearly understand concrete examples of how the skill graph will be used. This understanding can come from direct user examples or generated examples validated with user feedback.

## How to Gather Examples

Ask questions like:

- "What functionality should this skill support?"
- "Can you give some examples of how this skill would be used?"
- "I can imagine users asking for things like [example]. Are there other use cases?"
- "What would a user say that should trigger this skill?"

To avoid overwhelming users, limit to 3–5 questions per message. Start with the most important and follow up as needed.

## When to Conclude

This step is complete when there is a clear sense of:

1. The functionality the skill should support
2. The trigger phrases and contexts that should activate it
3. The expected outputs and behaviors
4. Edge cases or variations to handle
