# Best Practices Research

How to gather and apply best practices for a given domain when building a skill graph.

## What to Research

1. **How existing tools/frameworks in this domain are structured** — What do production systems look like?
2. **Community recommendations** — What does the developer community consider standard practice?
3. **Documentation patterns** — How do official docs organize information in this domain?
4. **Common architectures** — What architectural patterns are prevalent?

## How to Research

### Web Search

Use targeted searches to find:

- Official documentation for relevant tools and technologies
- Community guides, tutorials, and best-practice articles
- Reference implementations and example projects
- Stack Overflow discussions on architecture and design decisions

### Documentation Analysis

When examining documentation:

- Note how information is organized (by feature, by use case, by difficulty level)
- Identify the common vocabulary and terminology
- Observe what is considered prerequisite knowledge vs. what is explained

### Pattern Identification

Look for recurring patterns across multiple sources:

- Common file/directory structures
- Standard naming conventions
- Typical workflow sequences
- Standard configuration approaches

## Applying Findings to Graph Design

Research findings directly influence:

- **Decomposition** — How to split content mirrors how the domain organizes its knowledge
- **Terminology** — Reference files should use the domain's standard vocabulary
- **Scope** — Best practices reveal what a comprehensive skill should cover
- **Prioritization** — Community emphasis reveals what matters most
