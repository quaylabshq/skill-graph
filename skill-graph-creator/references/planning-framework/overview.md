# Planning Framework

This framework governs how to plan and track work during skill graph creation. Every skill graph creation must use structured planning.

## Core Rule

Always create a structured todo list before implementation. Track progress throughout. This is non-negotiable.

| Component | Purpose | Reference |

| Todo structure | How to create and manage task lists | [todo-structure.md](todo-structure.md) |

## When to Plan

- **At the start of every skill graph creation** — Create the initial todo list
- **After step 4 (Plan)** — Refine the todo list with specific file-level tasks
- **When scope changes** — Update the todo list to reflect new understanding
- **When tasks are completed** — Mark them done immediately, not in batches

## Planning Principles

1. **Plan before building** — Never start implementation without a task list
2. **Break down aggressively** — Each todo should represent a single, completable action
3. **Track in real-time** — Update status as you work, not after
4. **Be explicit** — Every file to create, every script to write gets its own todo
5. **Include verification** — Todo lists must include validation and testing steps
